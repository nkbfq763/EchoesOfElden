# ゲーム基本仕様書 (GameDesigner)

プロジェクト名: **Tales-like JRPG** (Godot Engine 4.7 / 2D)
作成: 設計エージェント (GameDesigner)
対象読者: Programmer / AssetManager / Tester

---

## 1. コンセプト

テイルズ オブ エターニア風の 2D アクション RPG。
フィールドはトップダウン（見下ろし型）で探索し、敵シンボルに接触すると
横スクロール型のアクションバトル（**リニアモーションバトル / LMB 風**）に切り替わる。

- プレイフィール: 軽快な移動 + テンポの良い連携攻撃 + 術技（スキル）
- ボリューム: まずは「町 → フィールド → ダンジョン1 → ボス1」の縦割り1本を完成させる
- 開発方針: Usage削減のため素材は外部AIで生成し、コードは最小限にまとめる

## 3. 世界観・ストーリー

- 大陸「**エターナ**」を舞台にした剣と魔法のファンタジー。
- 「精霊（スピリット）」の力を借りて術を使う設定。属性は 火 / 水 / 風 / 地 / 光 / 闇。
- 世界には少しずつ「異変」が広がりつつあり、封印が緩んで各地に魔物が溢れ出している。
- トーン: 明るく彩度高めのアニメ塗り。テイルズ系の「ポップだが芯のあるファンタジー」＋幼馴染×異世界の恋愛群像。

### あらすじ
- 物語の舞台は主人公たちが育った**開始地点の町**。主人公・幼馴染女・年上の青年の**幼馴染3人**が育った。
- 年上の青年は18歳の頃、少しずつ変わりゆく世界の異変にただ1人気付き、**世界を救う旅に単身で出発**した（現在22歳）。
- 物語開始時点：18歳の誕生日の日、町の近くの遺跡の状態を確認する試練に出かける。遺跡の中で眠ったままの少女に出会い、主人公が着けているお守りが反応し目を覚ましてしまう。
- 遺跡の女の子は自身の**目的を果たすため**に旅立ち、主人公も共にする。

### 恋愛関係（サブテーマ）
- 幼馴染女 → 主人公より年齢は１つ上で、主人公を子ども扱いしている。年上の青年に**憧れ**を抱く。同時に主人公への好意にも**本人はまだ気付いていない**。
- 遺跡の女の子 → 主人公になついている無口な女の子。
- この三角（四角）関係が旅の中で少しずつ描かれる。

### 主要キャラクター

| ID | 名前 | 年齢/性別 | クラス | 役割(戦闘) | 特徴・立ち位置 |
| --- | --- | --- | --- | --- | --- |
| `hero` | Roland Hartwell（ローランド・ハートウェル） | 18 / 男 | 剣士 | 前衛アタッカー(操作主体) | 幼馴染。遺跡の子を目覚めさせ旅が始まる |
| `childhood_girl` | Fiona Merrick（フィオナ・メリック） | 18 / 女 | 僧侶/術士 | 後衛サポート(回復/補助) | 年上青年に憧れ、主人公への好意は未自覚 |
| `otherworld_girl` | Selene Aurelis（セレネ・アウレリス） | 18 / 女 | 魔術士 | 中衛アタッカー(術) | 遺跡で目覚めた。固有の目的を持ち主人公に恋する |
| `elder_youth` | Gareth Vaughn（ガレス・ヴォーン） | 22 / 男 | 重戦士 | 前衛タンク/リーダー | 異変に最初に気付き単身旅立った先達。後に合流 |

> `character_name` はフルネーム、通常表示は名（Roland / Fiona / Selene / Gareth）を用いる。

## 4. 戦闘仕様（テイルズ オブ エターニア風 LMB）

### 4.1 基本ルール
- エンカウント: フィールド上の**敵シンボル接触**でバトルシーンへ遷移（シンボルエンカウント）。
- 戦場: 横1ラインのアリーナ。プレイヤーは左右移動 + ジャンプ。敵は複数体出現。
- 操作対象: 1体を直接操作、他の味方はAI（作戦で挙動を指定）。

### 4.2 アクション
| 入力 | 行動 | 説明 |
| --- | --- | --- |
| 左右 | 移動 | 敵との間合い調整 |
| ジャンプ | 空中回避 / 空中コンボ | |
| 通常攻撃(A) | 通常攻撃 | 最大3段の地上コンボ |
| 術技(B) | 特技/術 | TP消費。詠唱ありの術と即発の技 |
| ガード(L) | 防御 | 被ダメージ軽減 |
| ステップ(R) | 前後ステップ | 硬直キャンセル/回避 |

### 4.3 リソース
- **HP**: 0で戦闘不能。
- **TP** (Technical Points): 術技の消費リソース。通常攻撃ヒットで微回復。
- **コンボ数**: 連続ヒットでダメージ倍率アップ（演出）。

### 4.4 勝敗
- 敵全滅で勝利 → 経験値・ガルド（通貨）獲得 → フィールドに戻る。
- パーティ全滅で敗北 → タイトル/直近セーブへ。

### 4.5 ダメージ計算（初期式）
```
damage = max(1, attacker.attack * skill_power - defender.defense)
```
`skill_power` は通常攻撃=1.0、技/術ごとに係数を持つ。

## 5. マップ構造

```
[タイトル] → [オープニング(空から少女が落下→主人公が抱きとめる)] → [町(拠点)] ⇄ [フィールド] → [ダンジョン] → [ボス部屋]
                                                                        ↑                    ↓
                                                                    (店/宿/セーブ)      (敵シンボル→バトル)
```

| マップ | 種別 | 内容 |
| --- | --- | --- |
| タイトル | UI | New / Continue |
| オープニング | イベント | 主人公固定。空から `otherworld_girl` が落下→抱きとめる導入イベント（**キャラ選択は廃止**） |
| 町 (Town) | 探索 | 始まりの町（下記5.1）。NPC・店・宿・出口。安全地帯 |
| フィールド | 探索 | 敵シンボル配置、ダンジョン入口へ |
| ダンジョン | 探索 | 通路+部屋、宝箱、ギミック |
| ボス部屋 | バトル | ボス戦専用アリーナ |

- マップは **TileMapLayer** ベースで構築（現状の背景1枚絵+矩形コリジョンから移行）。
- マップ間遷移は「出入口エリア (Area2D)」で管理。

### 5.1 始まりの町「エルデン村」（仮）

- **コンセプト**: 大陸エターナ辺境の、緑と水に囲まれた静かな農村。まだ「異変」の影響が薄い、幼馴染3人が育った穏やかな故郷。移動チュートリアルを兼ねる。
- **雰囲気**: 石畳と土の道、赤茶の三角屋根、木の柵と花壇、中央に井戸のある広場。周囲は麦畑・草原・森、町外れに川と水車。明るく彩度高めのアニメ塗り（テイルズ系）。
- **規模/導線**: 640x360で約1〜1.5画面分の小さめマップ。迷わない一本道＋広場中心。**安全地帯（敵なし）**。
- **役割**: 移動チュートリアル → 導入イベント(Seleneの落下) → 南の出口からフィールドへ送り出す拠点。

**主なロケーション**:
| 場所 | 内容 | 実装フェーズ |
| --- | --- | --- |
| 中央広場（井戸） | 導入イベント発生地。Seleneが空から落下→Rolandが抱きとめる | P1(簡易) |
| Rolandの家 / Fionaの家 | 隣同士。幼馴染設定を表現 | P1(外観), P2(内部) |
| Garethの実家 | 旅立った先達の家（現在不在） | P2 |
| 村長の家 | 最初の目的/フックを渡すNPC | P2 |
| 道具屋 / 宿屋 | 建物のみ先行、機能はP2以降（宿=回復, 道具屋=購入, セーブ） | P1(外観), P2(機能) |
| 町の出口（南） | フィールドへの接続（Area2D遷移） | P1 |

> P1では建物・道・広場・出口の配置と歩行/衝突・出口遷移までを構築。NPC会話・店/宿機能・恋愛群像の布石はP2以降。
> 素材は町タイルセット(`town_tileset`)＋建物オブジェクト(`town_objects`)を使用（`01_asset_list.md`／`02_asset_prompts.md`）。P1は暫定タイル/単色で可。

## 8. 技術方針（Usage削減の指針）

- スプライトは**外部AIで生成**したスプライトシートを import して使う（DevinDesktop内で画像生成しない）。
  - 現状 `player.gd` はコードでピクセルを描画している。P3でAI素材へ差し替える前提だが、
    P1〜P2 は**プレースホルダ（単色矩形 or 既存procedural）**で先に挙動を作る。
- データ駆動: キャラ/敵/技は `Resource` (`.tres`) で定義し、コードを増やさない。
- シーンは責務ごとに分割（Field / Battle / UI）し、`autoload` で状態を共有。

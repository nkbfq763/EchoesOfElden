# 戦闘システム詳細仕様 — LMBS (Linear Motion Battle System)

作成: 設計エージェント (GameDesigner)
対象読者: Programmer / AssetManager / Tester
前提: `04_tales_reference.md`（エターニア分析）を踏まえた、テイルズ オブ エターニア風リアルタイム戦闘の詳細仕様。
方針: Usage削減のためコードは最小限・データ駆動。段階実装（B1〜B5）でまず「戦闘の芯」を作る。

> 現行バトルは本仕様の主要部分を実装済み。`battle.tscn`/`battle.gd`/`battle_unit.gd`にはリアルタイム移動、3段コンボ、TECH/CAST、ガード、ステップ、敵AI、報酬がある。味方AI、勝利演出、追加エフェクト、ボスAIは将来対応。

---

## 1. 設計思想（芯）
「**間合い管理 × 連携コンボ × 詠唱術の差し込み**」を面白さの中心に置く。
- 前衛が敵を拘束（コンボで怯ませる）→ 後衛が詠唱術を通す、という**役割分担**。
- 単発の殴り合いにしない。**通常→特技→奥義**の連携と、**距離の詰め引き**が主役。

---

## 2. 戦場（アリーナ）

- **サイドビュー・横1ライン**。奥行きは演出のみ（当たり判定はX距離＋高さYで扱う）。
- 画面: 640x360。地面ライン上に、左＝味方、右＝敵が対峙。
- **距離モデル**: キャラは地面ライン上をX方向に移動（左右）。ジャンプでY方向に一時離脱。
  - 「間合い」= 自キャラと対象敵とのX距離。近接技は近接間合い、術は遠距離から。
- 敵は1〜4体。複数はX方向に少しずつずらして配置し重なりを緩和。
- カメラは戦闘中は基本固定（全体が入る）。演出で軽くズーム可（P後半）。

---

## 3. バトルの状態機械（全体）

```
ACTIVE(リアルタイム) → RESULT(勝敗)
                              │
                    ┌─────────┴──────────┐
                (勝利: 敵全滅)        (敗北: 味方全滅)
                    │                     │
              報酬→元マップ復帰        ゲームオーバー/直近セーブ
```

- `BattleState` は `ACTIVE` / `RESULT`。`BATTLE_INTRO` は未実装。
- `ACTIVE` 中は**時間が常に流れる**（ターン制ではない）。全ユニットが並行して行動。
- ポーズ/メニュー（アイテム・術技選択）を開くと**時間停止**（P後半。B1は簡易でよい）。

---

## 4. ユニット（味方/敵共通）の状態機械

各キャラ/敵は `BattleUnit`（味方/敵共通のノード or クラス）として下記ステートを持つ:

```
IDLE ─ MOVE(前進/後退) ─ JUMP ─ ATTACK1/2/3 ─ TECH ─ CAST ─ GUARD ─ STEP ─ HURT ─ DOWN ─ KO
```

- 現在の`BattleUnit.State`は `IDLE, MOVE, JUMP, ATTACK1, ATTACK2, ATTACK3, TECH, CAST, GUARD, STEP, HURT, DOWN, KO`。
- `VICTORY` は未実装。
- 攻撃/技/詠唱中は**行動硬直**。硬直をステップや上位技で**キャンセル**できる余地を作る（連携の肝）。
- `HURT` はヒットで発生。連続ヒットで**のけぞり継続（コンボを受ける）**。一定以上で `DOWN`。
- `KO` = HP0。蘇生手段（術/アイテム）で復帰可（P後半）。

## 5. 操作キャラのアクション詳細

対象: プレイヤーが直接操作する1体（初期は `hero` = Roland）。

| アクション | 入力(暫定) | 内容 | コスト | 硬直/キャンセル |
| --- | --- | --- | --- | --- |
| 移動 | 左右 (A/D, ←→) | 敵へ前進/後退。間合い調整 | - | いつでも |
| ジャンプ | 上 or ジャンプ(Space) | 空中回避・空中攻撃の起点 | - | - |
| 通常攻撃 | `attack` (J) | 地上3段コンボ（1→2→3） | - | ステップ/特技でキャンセル可 |
| 特技 | `tech` (U) | TP消費の近接技。通常攻撃から連携で派生 | TP | 通常攻撃から連携で派生 |
| 術(アーツ) | `cast` (I) | 詠唱後に発動。詠唱中は無防備 | TP | 被弾で詠唱中断 |
| ガード | `guard` (K)長押し | 前方被ダメージ軽減。TP微回復 | - | 解除は任意 |
| ステップ | `step` (L) | 前後に素早く回避。硬直キャンセル | - | 攻撃硬直をキャンセル |
| 対象切替 | `target_next` (Tab) | 攻撃対象の敵を切替 | - | - |
| 操作キャラ切替 | (P後半) | パーティ内の操作キャラを変更 | - | - |

> 現在のInputMapは `attack`/`tech`/`cast`/`guard`/`step`/`jump`/`target_next`/`walk_mod`/`menu`。キーは順に J/U/I/K/L/Space/Tab/Shift/Escape・M。ゲームパッド対応は将来。

### 5.1 コンボ/連携ルール
- 通常攻撃は最大3段。各段のヒット確認で次段が出る（空振りでも段は進む）。
- **連携**: 通常攻撃(任意段) → 特技 → 奥義 の順で繋げると硬直を短縮して出せる。
- ステップで硬直をキャンセルし、再度コンボへ。
- コンボ数（連続ヒット）を画面に表示。ダメージに軽い倍率補正（例: 5hitごとに+数%、上限あり）。

---

## 6. リソース（HP / TP）

- **HP**: 0で `KO`。
- **TP**: 特技・術の消費。**通常攻撃ヒット/ガードで微回復**。休息(宿/アイテム)で全回復。
- 敵にもHPあり（TPは持たない or 内部管理）。
- ダメージ後の**現在値はバトル間で継続**（宿・回復で戻す）。全滅回避のリソース管理を促す。

---

## 7. ダメージ計算

```
base   = attacker.attack * skill_power
raw    = base - defender.defense
elem   = raw * element_multiplier          # 弱点1.5 / 半減0.5 / 無効0 / 等倍1.0
combo  = elem * (1 + min(combo_hits * 0.01, 0.3))   # 連携補正 上限+30%
damage = max(1, floor(combo) + randi_range(-5, 5))
```

- `skill_power`: 通常=1.0、特技/奥義/術ごとに係数（`SkillData.power`）。
- 属性: 現在は `none/fire/ice/lightning`。`water/wind/earth/light/dark` と敵ごとの弱点/耐性は将来対応。
- クリティカル（P後半, 任意）: 一定確率で1.5倍。

---

## 8. 敵AI

- 状態: `APPROACH(接近) → TELEGRAPH(予備動作) → ATTACK(攻撃) → RECOVER(後隙) → REPOSITION(再配置)` を巡回。
- **テレグラフ**: 攻撃前に短い予備動作（アニメ/エフェクト）を挟み、プレイヤーがガード/回避で反応できるようにする。
- 種類で挙動差: 近接（突進して殴る）、遠隔（距離を取って撃つ）、術（詠唱してから範囲）。
- 難度: `EnemyData` に `attack_interval` / `aggression` 等を持たせデータで調整。
- ボス: 複数パターン＋怒りモード（HP閾値で行動変化）。P後半。

---

## 9. 味方AI と 作戦（さくせん）

- 操作していない味方は**AIが自動行動**（近接は殴る、術者は回復/攻撃術）。
- **作戦**プリセットで方針指定（例: 「バランス」「TP温存」「術優先」「回復優先」「集中攻撃」）。
- 対象選択: プレイヤーの狙う敵に合わせる/自動で最寄り、等。
- B4で導入。B1〜B3は操作キャラ単体（ソロ）で骨組みを作る。

---

## 10. 勝敗・報酬・演出

- 勝利: 敵全滅 → **経験値・ガルド・アイテム**獲得 → 元マップへ復帰（撃破シンボルの扱いは §12 のエンカウント方式に依存）。
  - **勝利掛け合い**（キャラの一言＋勝利ポーズ）で“らしさ”を出す（B5）。
- 敗北: 味方全滅 → ゲームオーバー → タイトル/直近セーブ。
- 演出: ヒットストップ（被弾の一瞬停止）、斬撃/ヒットエフェクト、術の発動演出、コンボ表示、ダメージ数字ポップ。

## 11. バトルUI（HUD）

- 各キャラ: 名前 / HPバー / TPバー / 現在の作戦。
- コンボカウンタ、獲得ダメージ表示。
- 詠唱ゲージ（術発動までの進行）。
- 術技ショートカット（選択中の技/術）。
- ターゲットマーカー（狙っている敵の頭上）。
- 素材が来るまでは ProgressBar / Label / 単色マーカーの暫定UIで実装可。

## 12. エンカウント方式（要確定）

エターニア原典は**ランダムエンカウント**。本プロジェクトは実装容易さと現代的手触りから、既定を**シンボルエンカウント**（フィールド上の敵に接触で戦闘）として設計する。
- どちらでも `battle.tscn` の内部仕様（本書 §2〜§11）は共通。差は「戦闘へ入る導線」だけ。
- **要確認**: シンボル / ランダム / 併用（フィールドはシンボル、ダンジョンはランダム 等）のいずれにするか。

## 14. 必要アニメーション（アセット要件の要約）

戦闘で各ユニットに必要なアニメーション状態（`AnimationPlayer`/`AnimatedSprite2D` の名前案）。詳細な生成プロンプトは `06_battle_assets_pixellab.md`。

### 14.1 前衛（剣士=hero / 重戦士=elder_youth）
`idle, walk_fwd, walk_back, jump, attack1, attack2, attack3, tech, guard, step, hurt, down, victory`

### 14.2 後衛/術者（僧侶=childhood_girl / 魔術士=otherworld_girl）
`idle, walk_fwd, walk_back, cast_charge, cast_loop, cast_release, staff_swing, guard, hurt, down, victory`

### 14.3 敵（slime / goblin / boss ほか）
`idle, move, attack(1..n), hurt, defeat`（ボスは `enrage`, 複数attack）

### 14.4 エフェクト（キャラ非依存）
`slash, hit_spark, guard_spark, heal, cast_circle, 属性術(fire/water/wind/earth/light/dark)`

> 現行のRoland/Wolf入力素材は約124x124の個別PNGフレームで、表示時におおむね`scale=0.4`を適用する。64x64/128x128セル、専用hit/death/victory素材、エフェクトは将来の生成要件。基準向きは**右向き**（左は水平反転）。
